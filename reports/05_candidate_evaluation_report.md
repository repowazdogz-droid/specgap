## Limitations

- **Passing means no semantic divergence was detected within the abstract sandbox model and extracted constraints. It does not prove semantic correctness.**
- SpecGap does not prove the spec is correct. It identifies likely semantic divergence zones and checks extracted constraints (via Z3 over the abstract sandbox model) for consistency or implication failure.
- This is spec-conditioned candidate *evaluation* — a vericoding-adjacent workflow, not verified synthesis. SpecGap does not generate or repair the candidate policies; it only discriminates between candidates supplied to it.
- Candidate constraints are extracted with the same fixed-vocabulary rule extractor; unrecognized phrasing is silently not extracted, so a candidate could pass by omission rather than by correctness.
- The Z3 model is a small propositional abstraction of sandbox behavior, not a model of a real kernel or runtime.
- Counterexamples are illustrative behaviors, not exploit proofs.
- The candidate ordering reflects only implication success/failure against the extracted intent; it is not a quality, performance, or security score.
